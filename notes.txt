Important CLIs used for git and github:

-- git init (initialize a local repository)
-- git add . (adding all the modified file into stage area/ tracking area)
-- git add index.html (adding specific modified file e.g., index.html into stage area)
-- git commit -m "first init" (commit the changes from stage to local repository)

✅ Verify author of last commit:             -- git log -1
    To get the log of committed versions     -- git log 

-- git restore --staged index.html  (used to revert from stage to modified state)

**git by default create Master branch, in case if required to create manual branches :
    -- git branch -M main
    
    ** to check the branches    
        -- git branch

** Create a github account and add a new repository on cloud (github in this case)
    -- KohinoorPharmaGitRepo.git  is cloud repository.
    To connect the local repo wiht cloud repo use below command:
      -- git remote add origin https://github.com/shadabgit/KohinoorPharmaGitRepo.git
      -- git remote -v     (used to check the remote versions)
    
    It will pushes your local main branch to the remote repository named origin and sets up a tracking relationship (-u /upstream) between the two
      -- git push -u origin main

      Note: 
        -u (or --set-upstream): This flag is the key part that simplifies future Git operations. After using -u once for a branch, 
                            you can subsequently use simple commands like git push or git pull without having to specify origin main
                            every time. Git remembers the connection.


🔑 Priority order Git uses
        ** Repository (.git/config)
        ** Global (~/.gitconfig)
        ** System

🧠 Common issue (important):
        * Your Git username/email ≠ GitHub login email
        * Git email must match a verified email in GitHub for commits to link properly.


To get the root directory of the current Git repository from CLI, use this command:
    -- git rev-parse --show-toplevel  (Prints the absolute path of the Git repository’s root folder. Works from any subfolder inside the repo)

🔹 Extra useful variants
    Check if you are inside a Git repo:
        -- git rev-parse --is-inside-work-tree
    Get .git directory path:
        -- git rev-parse --git-dir

This works in Git Bash, CMD, PowerShell, and Linux/macOS terminals.


Use these CLI commands to verify Git user details 👇 -----------------------------

✅ Check user configured for the current Git repository
(Repo-specific config)
    git config user.name
    git config user.email

✅ Check global Git user (used by default)
(System-wide)
    git config --global user.name
    git config --global user.email

✅ See which user is actually active for this repo
(Git applies repo → global → system priority)
    git config --show-origin --get user.name
    git config --show-origin --get user.email

    ** This shows where the value is coming from (.git/config or ~/.gitconfig).

✅ List ALL Git config values (filtered)
        git config --list | findstr user


✅ Open repository-level Git config (from inside a Git repo)    
    -- git config --edit
    ✔ Opens .git/config in your default editor e.g., in my case the local dir .git config file.

✅ Open global Git config (user-wide)
    -- git config --global --edit      (in my case [user]  name = shadabgit    email = wasai.gimt@gmail.com)
